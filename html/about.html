About This Project

Software Engineering | Semester Project

What is this?

This web application demonstrates how monitoring tools — specifically Logs, Metrics, and Alerts — reduce corrective maintenance response time in software systems.

When a bug or failure occurs, monitoring tools detect it instantly, notify the team, and provide diagnostic information — drastically reducing the time it takes to identify and fix the issue.

The Three Monitoring Tools

📋

Logs

Logs record every event in the system with timestamps. When an error occurs, logs show exactly what happened, where, and when — making diagnosis fast.

📊

Metrics

Metrics track numerical values like CPU usage, response time, and error rate. Sudden spikes indicate a problem before users even notice it.

🔔

Alerts

Alerts automatically notify engineers when a metric crosses a threshold. This eliminates manual checking and ensures immediate response.
